发表于 2016-07-10   |   分类于 技术   |  

使用工厂方法原因

  在Java中，新建对象是通过修饰符new + "className"来实现的。但是这样存在一个问题，使用new实例化一个具体类是使用了 实现，而非接口。当加入新的具体类的时候，就必须修改代码，违反了 开闭原则。我们将新建具体类的代码封装到一个类里面，这个类就是工厂(fatory)。

工厂方法的分类

  工厂方法模式分为三种：简单工厂模式、工厂方法模式、静态工厂模式，其中用的最好的是静态工厂模式

简单工厂模式

概述

  定义一个工厂类，它可以根据参数的不同返回不同类的实例，被创建的实例通常都具有共同的父类。

简单工厂模式

//title 简单工厂模式
class AbstractProduct
class ConcreteProductA
class ConcreteProductB
class Factory{
  +AbstractProduct produce(String type);


}
note right of Factory : public AbstractProduct produce(String type)     {\nAbstractProduct product = null;\nif ("1".equals(type)) {\nproduct = new  ConcreteProductA();\n} else if ("2".equals(type)) {\nproduct = new   ConcreteProductB();\n}\nreturn product;\n}
AbstractProduct<|-down- ConcreteProductB
AbstractProduct<|-down- ConcreteProductA
Factory..>AbstractProduct

工厂方法模式

//title 工厂方法模式
class AbstractProduct
class ConcreteProductA
class ConcreteProductB
class Factory{
  +public AbstractProduct produceProductA();
  +public AbstractProduct produceProductB();


}
note right of Factory : public AbstractProduct produceProductB()     {\nreturn new ConcreteProductB();\n}\npublic AbstractProduct produceProductA()     {\nreturn new ConcreteProductA();\n}
AbstractProduct<|-down- ConcreteProductB
AbstractProduct<|-down- ConcreteProductA
Factory..>AbstractProduct

//title 静态工厂模式

class ConcreteProductA
class ConcreteProductB
class Factory{
  {static} +public AbstractProduct produceProductA();
  {static} +public AbstractProduct produceProductB();


}
note right of Factory :  public static AbstractProduct produceProductB()     {\nreturn new ConcreteProductB();\n}\npublic staticAbstractProduct produceProductA()     {\nreturn new ConcreteProductA();\n}
